top-left rule

改了什么
以前在 raster.cpp (line 146) 那段，像素是否落在三角形里，是用像素中心 (i + 0.5, j + 0.5) 去测的；但一旦判定命中，后面的 UV/颜色/深度 插值却是按整数像素点 (i, j) 算。[旧逻辑对应现在的插值入口是 raster.cpp (line 211)] 这会导致“这个像素属于谁”和“这个像素该采什么颜色”不是同一个采样点。矩形拆成两个三角形时，公共对角线两边就很容易不连续。

现在改成了：

用 edge_function() 统一表达点在边哪一侧。raster.cpp (line 9)
覆盖测试和插值都用同一个像素中心 (px, py)。raster.cpp (line 163) raster.cpp (line 220)
插值函数签名也改成显式接收 sample_x/sample_y，避免再偷偷用整数像素角点。raster.h (line 35)
为什么 top-left rule 有用
两个三角形共享一条边时，必须规定“边界像素到底归左边三角形，还是右边三角形”。如果不规定，公共边上的像素就会：

要么两边都画，透明混合时变暗
要么两边都不画，直接露出黑底
top-left rule 就是在边函数接近 0 的时候，给边界一个稳定归属规则：只有“top 或 left”的边可以包含边界点，另一侧不包含。raster.cpp (line 15) raster.cpp (line 177)

这样两个共享边的三角形不会同时声称“这个像素归我”，也不会同时放弃它。

边函数有方向性，顺时针和逆时针三角形算出来的符号会反过来。所以现在先算一次三角形总面积符号：

如果是负的，就把三个边函数都翻回正方向再做 inside test。raster.cpp (line 149) raster.cpp (line 170)
这一步的意义是：不管顶点顺序怎么来，覆盖规则都一致，不会因为 winding 不同导致一半矩形正常、一半矩形有缝。

为什么旧实现容易出黑缝
旧实现有两个典型问题：

覆盖点和插值点不一致
重心权重用 abs(area) 这类对称写法，方向信息被抹掉
结果就是公共边两侧虽然“都在矩形里”，但采样出来的 UV/颜色 不严格连续。对纯色矩形也许只是很淡的一条线；对贴图或半透明 UI，黑底会被放大成明显的三角分割线。

一句话总结：这次不是简单“修 bug”，而是把光栅器改回了更标准的做法：
edge function 做覆盖，top-left rule 管共享边归属，pixel center 同时用于覆盖和插值。这样矩形拆成两个三角形时，算法上才真正是连续的。
